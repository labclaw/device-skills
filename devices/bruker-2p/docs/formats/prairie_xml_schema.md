# Prairie View XML Metadata Schema

> Source: PrairieView file format documentation, DataJoint PrairieViewMeta parser,
> and analysis of XML files generated by PrairieView 5.x.

## Overview

PrairieView stores acquisition metadata in XML files alongside the image data. Every
acquisition generates an XML file containing complete information about scan parameters,
hardware state, timing, and file locations.

**Critical parsing note:** PrairieView writes `<?xml version="1.1"?>` headers, but
Python's `xml.etree.ElementTree` only supports XML 1.0. You **must** use `lxml.etree`
or strip the declaration before parsing. See the parsing section below.

## File Naming

XML metadata files follow the same naming convention as image files:

```
TSeries-DDMMYYYY-HHMM-###.xml     # T-series metadata
ZSeries-DDMMYYYY-HHMM-###.xml     # Z-stack metadata
SingleImage-DDMMYYYY-HHMM-###.xml  # Single frame metadata
```

## Three-Level Hierarchy

Prairie XML uses a three-level structure:

```
PVScan (root)
├── PVStateShard (global state at scan start)
│   └── PVStateValue (repeated, one per parameter)
├── Sequence (one per time point, Z-plane, or cycle)
│   ├── PVStateShard (state at this sequence point)
│   │   └── PVStateValue (parameter changes from global)
│   └── Frame (one per frame in this sequence)
│       ├── PVStateShard (state at this frame)
│       │   └── PVStateValue (parameter changes from sequence)
│       ├── File (image file reference)
│       │   ├── @channel: channel number
│       │   ├── @channelName: channel name string
│       │   └── @filename: relative path to OME-TIFF file
│       └── ExtraParameters (timing, indices)
│           ├── @lastGoodFrame: index of last valid frame
│           └── @framePeriod: time between frames (seconds)
```

### PVScan (Root Element)

| Attribute | Type | Description |
|-----------|------|-------------|
| `version` | string | PrairieView version (e.g., "5.6.64.400") |
| `date` | string | Acquisition start date |
| `notes` | string | User-entered notes |
| `sequence_type` | string | "TSeries", "ZSeries", "Single", etc. |

### Sequence Element

| Attribute | Type | Description |
|-----------|------|-------------|
| `type` | string | Sequence type identifier |
| `cycle` | int | Cycle number (1-based) |
| `time` | float | Time offset from scan start (seconds) |

### Frame Element

| Attribute | Type | Description |
|-----------|------|-------------|
| `relativeTime` | float | Time relative to sequence start (seconds) |
| `absoluteTime` | float | Time relative to scan start (seconds) |
| `index` | int | Frame index within the sequence |
| `parameterSet` | string | Parameter set identifier |

## PVStateValue Keys

These are the parameter keys found in `PVStateValue` elements. The `@key` attribute
identifies the parameter, and `@value` contains its value. Some parameters use
`IndexedValue` sub-elements for multi-dimensional values.

### Scan Parameters

| Key | Type | Example | Description |
|-----|------|---------|-------------|
| `activeMode` | string | "ResonantGalvo" | Current scan mode |
| `pixelsPerLine` | int | "512" | X resolution |
| `linesPerFrame` | int | "512" | Y resolution |
| `micronsPerPixel` | indexed | X: "1.56", Y: "1.56" | Pixel size |
| `opticalZoom` | float | "1.0" | Zoom factor |
| `dwellTime` | float | "0.2" | Pixel dwell time (us) |
| `framePeriod` | float | "0.0333" | Frame period (seconds) |
| `scanLinePeriod` | float | "0.0000625" | Line period (seconds) |
| `rastersPerFrame` | int | "1" | Rasters per frame |
| `resonantScanFreq` | float | "7910" | Resonant frequency (Hz) |

### Laser Parameters

| Key | Type | Example | Description |
|-----|------|---------|-------------|
| `laserPower` | indexed | laser 0: "25.0" | Pockels setting (%) per laser |
| `laserWavelength` | indexed | laser 0: "920.0" | Wavelength (nm) per laser |

## XML 1.0 vs 1.1 Parsing Issue

### The Problem

PrairieView generates XML files with this declaration:

```xml
<?xml version="1.1" encoding="utf-8"?>
```

Python's built-in `xml.etree.ElementTree` only supports XML 1.0 and will raise:

```
xml.etree.ElementTree.ParseError: XML or text declaration not at start of entity
```

or silently fail to parse.

### Solution 1: Use lxml (Recommended)

```python
from lxml import etree

# Parse with recovery mode to handle any malformed elements
tree = etree.parse("TSeries-01012024-1200-001.xml",
                   parser=etree.XMLParser(recover=True))
root = tree.getroot()

# Access PVStateValues
for sv in root.iter("PVStateValue"):
    key = sv.get("key")
    value = sv.get("value")
    if value:
        print(f"{key} = {value}")
    else:
        # Multi-valued: check IndexedValue sub-elements
        for iv in sv.findall("IndexedValue"):
            idx = iv.get("index") or iv.get("description")
            val = iv.get("value")
            print(f"{key}[{idx}] = {val}")
```

### Solution 2: Strip XML Declaration

```python
import xml.etree.ElementTree as ET

with open("TSeries-01012024-1200-001.xml", "r") as f:
    content = f.read()

# Remove the XML declaration line
if content.startswith("<?xml"):
    content = content[content.index("?>") + 2:].lstrip()

root = ET.fromstring(content)
```

### Solution 3: Replace Version

```python
content = content.replace('version="1.1"', 'version="1.0"')
root = ET.fromstring(content)
```

## Complete Parsing Example

```python
from lxml import etree
from pathlib import Path


def parse_prairie_xml(xml_path: str) -> dict:
    """Parse a PrairieView XML metadata file into a structured dictionary.

    Args:
        xml_path: Path to the .xml metadata file.

    Returns:
        Dictionary with scan metadata, frame info, and file references.
    """
    tree = etree.parse(xml_path, parser=etree.XMLParser(recover=True))
    root = tree.getroot()

    metadata = {
        "version": root.get("version"),
        "date": root.get("date"),
        "sequence_type": root.get("sequence_type", "Unknown"),
        "state": {},
        "sequences": [],
    }

    # Parse global PVStateValues
    for shard in root.findall("PVStateShard"):
        for sv in shard.findall("PVStateValue"):
            key = sv.get("key")
            value = sv.get("value")
            if value is not None:
                metadata["state"][key] = value
            else:
                indexed = {}
                for iv in sv.findall("IndexedValue"):
                    idx = iv.get("index") or iv.get("description", "0")
                    indexed[idx] = iv.get("value")
                metadata["state"][key] = indexed

    # Parse Sequences and Frames
    for seq in root.findall("Sequence"):
        seq_data = {
            "type": seq.get("type"),
            "cycle": seq.get("cycle"),
            "time": seq.get("time"),
            "frames": [],
        }
        for frame in seq.findall("Frame"):
            frame_data = {
                "index": frame.get("index"),
                "relative_time": frame.get("relativeTime"),
                "absolute_time": frame.get("absoluteTime"),
                "files": [],
            }
            for f in frame.findall("File"):
                frame_data["files"].append({
                    "channel": f.get("channel"),
                    "channel_name": f.get("channelName"),
                    "filename": f.get("filename"),
                })
            seq_data["frames"].append(frame_data)
        metadata["sequences"].append(seq_data)

    return metadata


# Usage
meta = parse_prairie_xml("TSeries-01012024-1200-001/TSeries-01012024-1200-001.xml")
print(f"Scan type: {meta['sequence_type']}")
print(f"Frames: {sum(len(s['frames']) for s in meta['sequences'])}")
print(f"Pixel size: {meta['state'].get('micronsPerPixel')}")
```
